1. Pips (Percentage in Point)
A pip is the smallest price move a currency pair can make.

For most pairs, 1 pip = 0.0001 (except JPY pairs where 1 pip = 0.01).

Example:
If EUR/USD moves from 1.1050 to 1.1051, that’s 1 pip.

Why it matters:
Pips help you measure profit or loss in trades.

2. Spread
The spread is the difference between the buy (ask) and sell (bid) price.

It's how brokers make money (especially commission-free brokers).

Example:
If EUR/USD is quoted as 1.1050/1.1052, the spread is 2 pips.

Why it matters:
The wider the spread, the more price must move in your favor before you're in profit.

3. Leverage
Leverage lets you control a large trade with a small amount of capital.

Expressed as a ratio like 1:50, 1:100, or 1:500.

Example:
With 1:100 leverage, $100 in your account can control $10,000 worth of trades.

Why it matters:

High leverage = higher profit potential, but also higher risk of blowing your account.

Use risk management (e.g., stop-loss, position sizing).

Bonus Concept: Lot Sizes
Standard lot = 100,000 units

Mini lot = 10,000 units

Micro lot = 1,000 units

1 pip =

$10 per pip (standard lot)

$1 per pip (mini lot)

$0.10 per pip (micro lot)

🌱 Beginner-Friendly Guide to .env Files and python-dotenv
🛡️ Why Use .env Files?
When developing applications, especially those involving sensitive information like API keys, database credentials, or secret tokens, it's crucial to keep this data secure and separate from your main codebase. Hardcoding such information directly into your scripts can lead to security vulnerabilities, especially if the code is shared or uploaded to platforms like GitHub.

A .env file allows you to store these sensitive details in a separate, non-public file. This approach:

Enhances Security: Keeps sensitive data out of your main codebase.

Improves Flexibility: Easily change configurations without modifying code.

Supports Multiple Environments: Manage different settings for development, testing, and production.

📄 What Is a .env File?
A .env file is a simple text file containing key-value pairs, each representing an environment variable. For example:

ini
Copy
Edit
API_KEY=your_api_key_here
DATABASE_URL=postgres://user:password@localhost:5432/mydatabase
SECRET_KEY=your_secret_key
DEBUG=True
Important: Ensure that your .env file is included in your .gitignore to prevent it from being tracked by version control systems like Git.

⚙️ Setting Up python-dotenv
To utilize the .env file in your Python project, follow these steps:

Install the python-dotenv Package:

Open your terminal or command prompt and run:

bash
Copy
Edit
pip install python-dotenv
Create a .env File:

In your project's root directory, create a file named .env and add your environment variables as shown above.

Load Environment Variables in Your Python Script:

In your Python script, use the following code to load and access the variables:

python
Copy
Edit
import os
from dotenv import load_dotenv

# Load variables from .env file
load_dotenv()

# Access variables
api_key = os.getenv('API_KEY')
database_url = os.getenv('DATABASE_URL')
secret_key = os.getenv('SECRET_KEY')
debug_mode = os.getenv('DEBUG')

print(f"API Key: {api_key}")
print(f"Database URL: {database_url}")
print(f"Secret Key: {secret_key}")
print(f"Debug Mode: {debug_mode}")
This script will output the values you've set in your .env file.

🧪 Practical Example: Using .env in a Flask Application
If you're developing a Flask web application, you can integrate the .env file as follows:

python
Copy
Edit
from flask import Flask
from dotenv import load_dotenv

@app.route('/')

if __name__ == '__main__':
    app.run(debug=True)
In this example, the Flask app retrieves the SECRET_KEY from the .env file, enhancing security by not exposing it directly in the code.

WEEK 2 NOTES: TYPES OF FOREX ORDERS (Beginner-Friendly)
1. What Is an Order?
An order is how you tell your broker to buy or sell a currency pair.

Main Types of Forex Orders You Need to Know
A. Market Order
What it does: Buys or sells immediately at the current price.

When to use: You want to enter or exit fast.

Example: You think EUR/USD will go up now → place a buy market order.

Note: No control over the exact price—you take whatever the market offers.

B. Limit Order
What it does: Buys or sells only if the price reaches a better level.

When to use: You want to wait for a better deal.

Types:

Buy Limit: Buy lower than current price.

Sell Limit: Sell higher than current price.

Example: EUR/USD is at 1.1000, but you only want to buy at 1.0950 → set a buy limit at 1.0950.

C. Stop Order
What it does: Triggers a market order after a certain price is hit.

When to use: You want to buy/sell on momentum or breakout.

Types:

Buy Stop: Buy when price goes above current level.

Sell Stop: Sell when price drops below current level.

Example: EUR/USD is at 1.1000. If it breaks to 1.1050, you want to join the trend → set a buy stop at 1.1050.

Essential Risk Management Orders
D. Stop-Loss (SL)
What it does: Automatically closes your trade if it goes against you.

Why it’s vital: Limits your losses. Keeps emotions out.

Example: You bought EUR/USD at 1.1000. You want to risk 50 pips → SL = 1.0950.

E. Take-Profit (TP)
What it does: Closes your trade automatically when it hits your target profit.

Why it’s useful: Locks in profits and reduces greed.

Example: You bought at 1.1000. Want 100 pip profit → TP = 1.1100.

Summary Table:
Order Type	Action	Use It When…
Market Order	Buy/sell instantly	You want to act fast
Limit Order	Buy/sell at a better price	You want a discount or higher price
Stop Order	Buy/sell when price breaks a level	You’re chasing momentum
Stop-Loss	Close trade if losing	You want to protect your capital
Take-Profit	Close trade at profit	You want to secure your gains
